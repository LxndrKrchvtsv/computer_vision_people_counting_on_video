## Инструкция по запуску (ВАЖНО)

Виртуальное окружение `.venv` использует Python **3.14**, а системный `py` — 3.13,
где не установлен `cv2`. Поэтому **запуск только через интерпретатор из `.venv`**:

```powershell
# Из корня проекта C:\Users\akric\PycharmProjects\cv_project
$env:PYTHONIOENCODING='utf-8'
.venv\Scripts\python.exe main.py --video data/test.mp4 --display
```

Либо сначала активировать окружение, затем `python`:

```powershell
.venv\Scripts\Activate.ps1
python main.py --video data/test.mp4 --display
```

НЕ запускать через `py main.py ...` — будет `ModuleNotFoundError: No module named 'cv2'`.

Тестовое видео: `data/test.mp4` — 1080×1920, 30 fps, 1764 кадра (~59 с), плотный
пешеходный переход.

## Диагноз проблемы (почему квадраты мелькают / счёт недостоверен)

1. **Классика (MOG2 + контуры) непригодна для плотной толпы**: силуэты сливаются
   в один блоб, зебра и шум дают ложные срабатывания, стоящие люди исчезают.
2. **Главная причина мелькания — самописный `CentroidTracker`**: ID к кадру 600
   доходят до #1300+ (огромная текучка). Жадное сопоставление только по дистанции,
   `max_distance=80` px на кадре высотой 1920, `max_missing=5`, нет предсказания
   движения → перекрытие = потеря ID = новый бокс = мелькание.
3. **YOLOv8n пропускает мелких/дальних людей** на полном кадре 1080×1920.
4. **Счёт недостоверен by design**: при потере/смене ID у линии возможен двойной
   подсчёт или недосчёт; нет ground truth для проверки.
