# **Мастер-документ проекта: Подсчет людей и оценка потока (Команда 3 человека)**

Данный документ является единой базой знаний для вашей команды. Здесь собраны требования к проекту, распределение ролей и готовые шаблоны (промпты) для запросов к нейросетям (ChatGPT/Claude/Gemini) для каждой стадии разработки.

## **1\. Базовая информация о проекте**

* **Тема:** 14\. Human Sys. People Counting & Flow (Подсчет людей и оценка направления движения).  
* **Задача:** Подсчет людей и оценка направления их перемещения на видео или последовательности кадров.  
* **Обязательное условие:** Полностью автоматическая система (от сырого кадра до финального решения).

## **2\. Обязательный Пайплайн и Распределение Ролей**

Система должна работать по строгому пайплайну: image → enhance → segment → clean → detect → decision. Так как вас 3 человека, роли распределены следующим образом:

| Роль | Ответственный | Стадии Пайплайна | Описание Задач |
| :---- | :---- | :---- | :---- |
| **Lead CV Engineer** | Витя | Detect \+ Decision \+ Data (Совмещено) | Сбор датасета, поиск контуров/боксов вокруг людей, логика подсчета (пересечение линий), сборка всего кода в единый репозиторий. |
| **Image Processing Specialist** | Александр | Enhance \+ Segment | Эквализация гистограммы, гамма-коррекция, вычитание фона (Background Subtraction) для выделения движущихся объектов. |
| **Morphology & Report Lead** | Богдан | Clean \+ Visualization \+ Report | Морфологическая очистка масок (эрозия, дилатация), отрисовка результатов (боксы, счетчики) на кадрах, написание финального PDF-отчета на 5–8 страниц. |

## **3\. Шпаргалка промптов для работы с Нейросетями**

Используйте эти промпты как стартовые шаблоны при обращении к ИИ для написания кода. Скопируйте нужный промпт и подставьте свои детали.

### **Для Студента 2 (Enhance \+ Segment)**

"Я пишу пайплайн на Python \+ OpenCV для трекинга людей с камеры наблюдения.  
1\. Напиши функцию для стадии Enhance: мне нужно выровнять освещение на кадрах. Предложи 2 варианта (например, CLAHE) и напиши код для их применения к кадру (numpy array).  
2\. Напиши функцию для стадии Segment: используй Background Subtraction (например, cv2.createBackgroundSubtractorMOG2), чтобы получить бинарную маску движущихся людей из последовательности кадров."

### **Для Студента 3 (Clean \+ Visualization)**

"У меня есть бинарная маска движущихся людей, полученная через MOG2. На ней много мелкого шума и дыр внутри силуэтов.  
1\. Напиши код на OpenCV для стадии Clean: примени морфологические операции (Opening для удаления шума, Closing для заполнения дыр). Напиши гибкую функцию, где я смогу менять размер ядра (kernel size).  
2\. Напиши функцию Visualization: она должна принимать оригинальный кадр, список bounding boxes и текущие счетчики (In/Out), и красиво отрисовывать прямоугольники и текст на кадре с помощью cv2.rectangle и cv2.putText."

### **Для Студента 1 (Detect \+ Decision \+ Integration)**

"У меня есть чистая бинарная маска движущихся объектов.  
1\. Напиши код для стадии Detect: используй cv2.findContours, отфильтруй контуры по минимальной площади и верни координаты центроидов (центров масс) для каждого человека.  
2\. Напиши логику для стадии Decide: предложи простой алгоритм трекинга центроидов между кадрами (по дистанции) и напиши код, который увеличивает счетчик 'In', если центроид пересек горизонтальную линию y=300 сверху вниз, и 'Out' — если снизу вверх."

## **4\. Чек-лист финальной сдачи (Deliverables)**

1. **Код:** Общий репозиторий (GitHub/GitLab/zip). Код должен запускаться без ошибок. Вклады каждого участника должны быть видны через коммиты.  
2. **Вывод системы:** Для каждого тестового видео/кадра алгоритм должен сохранять: оригинал, улучшенное фото, сырую маску, чистую маску, результат детекции и финальное решение.  
3. **Отчет:** Один PDF-документ на команду (5–8 страниц). Включает: описание проблемы, роли, дизайн пайплайна, методы, результаты по стадиям, примеры неудач (failure cases).  
4. **Презентация (10 мин):** Каждый участник защищает свою часть. Обязательна демонстрация работы (live demo).  
5. **Contribution Statement:** 1 страница с подписями всех участников, ролями, списком выполненных задач и согласованным распределением баллов.
